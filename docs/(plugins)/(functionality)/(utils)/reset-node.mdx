## Features

- Reset block formatting to paragraph on key events (Enter/Backspace).
- Built into the core editor behavior - no separate plugin needed.
- Configure reset behavior per node type using simple properties.

</PackageInfo>

## Kit Usage

<Steps>

### Installation

Reset node functionality is built into the core editor via `SlateExtensionPlugin`. No additional installation needed.

### Configure Plugins

Configure reset behavior on individual plugins using node properties:

```tsx
import { BlockquotePlugin, HeadingPlugin } from '@udecode/plate';
import { createPlateEditor } from '@udecode/plate/react';

const editor = createPlateEditor({
  plugins: [
    // ...otherPlugins,
    BlockquotePlugin.configure({
      node: {
        resetOnEmpty: true, // Reset to paragraph when empty + Enter/Backspace
        resetOnStart: true, // Reset when Backspace at start
      },
    }),
    HeadingPlugin.configure({
      node: {
        resetOnEmpty: true,
        resetOnStart: true,
      },
    }),
  ],
});
```

</Steps>

## Manual Usage

<Steps>

### Configure Node Properties

Add reset behavior to custom plugins:

```tsx
import { createSlatePlugin } from '@udecode/plate';

const CustomBlockPlugin = createSlatePlugin({
  key: 'customBlock',
  node: {
    isElement: true,
    resetOnEmpty: true, // Reset when block is empty and Enter/Backspace pressed
    resetOnStart: true, // Reset when Backspace pressed at block start
  },
});
```

### Special Cases

For complex reset logic, use `editor.api.shouldReset`:

```tsx
import { CodeBlockPlugin } from '@udecode/plate';

const editor = createPlateEditor({
  plugins: [
    // ...otherPlugins,
    CodeBlockPlugin.extendApi(({ editor }) => ({
      shouldReset: (trigger) => {
        if (trigger === 'empty') {
          return editor.api.isCodeBlockEmpty();
        }
        if (trigger === 'start') {
          return editor.api.isSelectionAtCodeBlockStart();
        }
        return false;
      },
    })),
  ],
});
```

</Steps>

## Node Properties

Configure reset behavior using these node properties:

### `resetOnEmpty`

**Type:** `boolean`

When `true`, pressing Enter or Backspace in an empty block will reset it to a paragraph.

```tsx
const BlockquotePlugin = createSlatePlugin({
  key: 'blockquote',
  node: {
    isElement: true,
    resetOnEmpty: true,
  },
});
```

### `resetOnStart`

**Type:** `boolean`

When `true`, pressing Backspace at the start of a block will reset it to a paragraph while preserving content.

```tsx
const HeadingPlugin = createSlatePlugin({
  key: 'h1',
  node: {
    isElement: true,
    resetOnStart: true,
  },
});
```

## API

### `editor.api.shouldReset`

Override reset behavior for complex cases.

<API name="shouldReset">
<APIOptions>
  <APIItem name="trigger" type="'empty' | 'start'">
    The trigger that initiated the reset check.
    - `'empty'`: Triggered by Enter/Backspace in empty block
    - `'start'`: Triggered by Backspace at block start
  </APIItem>
</APIOptions>

<APIReturns type="boolean">
  Whether the block should be reset to paragraph.
</APIReturns>
</API>

## Examples

### Basic Configuration

Most blocks can use simple boolean configuration:

```tsx
const editor = createPlateEditor({
  plugins: [
    // Reset headings when empty or at start
    HeadingPlugin.configure({
      node: { resetOnEmpty: true, resetOnStart: true },
    }),
    
    // Reset blockquotes when empty or at start  
    BlockquotePlugin.configure({
      node: { resetOnEmpty: true, resetOnStart: true },
    }),
    
    // Reset callouts when empty or at start
    CalloutPlugin.configure({
      node: { resetOnEmpty: true, resetOnStart: true },
    }),
  ],
});
```

### Advanced Configuration

For complex logic, extend the API:

```tsx
const CodeBlockPlugin = createSlatePlugin({
  key: 'codeblock',
  node: {
    isElement: true,
    // Don't use simple boolean - use custom logic instead
  },
})
.extendApi(({ editor }) => ({
  shouldReset: (trigger) => {
    if (trigger === 'empty') {
      // Custom logic for empty code blocks
      return editor.api.isCodeBlockEmpty();
    }
    if (trigger === 'start') {
      // Custom logic for code block start
      return editor.api.isSelectionAtCodeBlockStart();
    }
    return false;
  },
}));
```

## Migration

### From ResetNodePlugin

**Before (old API):**
```tsx
ResetNodePlugin.configure({
  options: {
    rules: [
      {
        types: ['blockquote', 'h1', 'h2'],
        defaultType: 'p',
        hotkey: 'Enter',
        predicate: (editor) => editor.api.isEmpty(editor.selection, { block: true }),
      },
      {
        types: ['blockquote', 'h1', 'h2'], 
        defaultType: 'p',
        hotkey: 'Backspace',
        predicate: (editor) => editor.api.isAt({ start: true }),
      },
    ],
  },
})
```

**After (new API):**
```tsx
// Configure each plugin individually
BlockquotePlugin.configure({
  node: { resetOnEmpty: true, resetOnStart: true },
}),
HeadingPlugin.configure({
  node: { resetOnEmpty: true, resetOnStart: true },
}),
```

The new approach is simpler and more maintainable - configure reset behavior directly on each plugin rather than centralized rules.
