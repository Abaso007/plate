Plate 提供多种集成方式，适用于不同项目需求和环境。请选择最符合你需求的集成路径。

## 客户端集成

### 方式一：使用 Plate UI（推荐）

如果你正在使用或计划使用 [shadcn/ui](https://ui.shadcn.com/)，推荐通过 [Plate UI](/docs/plate-ui) 获得丰富、预配置的编辑器体验。

<Steps>

#### 安装 Plate UI

首先，参考 **[Plate UI 安装指南](./installation/plate-ui)**，完成核心依赖安装和 Plate UI 组件配置。

#### 基本用法

Plate UI 配置完成后，根据你的 React 应用类型选择对应指南：

*   **[Next.js](./installation/next)** - 适用于服务端渲染应用
*   **[React](./installation/react)** - 适用于客户端应用（Vite、React Router 等）

</Steps>

### 方式二：手动集成

如果你的项目未使用 shadcn/ui、Tailwind CSS，或需要框架无关的解决方案，请参考 **[手动安装指南](./installation/manual)** 获取详细步骤。

<Callout type="note">
  你可以随时在手动集成基础上引入 Plate UI 组件。
</Callout>

## 服务端集成

### React Server Components

如需在 React Server Components (RSC) 中使用 Plate，实现静态内容生成或服务端数据处理（无需客户端交互），请参考 **[RSC 指南](./installation/rsc)**。

### Node.js 环境

如需在 Node.js 环境下使用 Plate 进行后端数据处理、内容迁移或校验，请参考 **[Node.js 指南](./installation/node)**。
