---
title: 路径（Path）
description: Slate 中路径的 API 参考文档。
---

路径（Path）是一个索引列表，描述了节点在 Slate 节点树中的确切位置。虽然它们通常是相对于根 `Editor` 对象的，但它们可以相对于任何 `Node` 对象。

```ts
type Path = number[];
```

## `PathApi`

### `operationCanTransformPath`

检查一个操作是否会影响路径（用作规范化过程中脏路径更新的优化）。

<API name="operationCanTransformPath">
<APIParameters>
  <APIItem name="operation" type="Operation<N>">
    要检查的操作。
  </APIItem>
</APIParameters>

<APIReturns type="boolean">
  如果操作是插入、合并、移动、删除或分割操作，则返回 `true`。
</APIReturns>
</API>

### `transform`

通过操作转换路径。

<API name="transform">
<APIParameters>
  <APIItem name="path" type="Path">
    要转换的路径。
  </APIItem>
  <APIItem name="operation" type="Operation">
    要应用的操作。
  </APIItem>
  <APIItem name="options" type="PathTransformOptions" optional>
    转换路径的选项。
  </APIItem>
</APIParameters>

<APIOptions type="PathTransformOptions">
  <APIItem name="affinity" type="TextDirection | null" optional>
    转换的亲和性。
  </APIItem>
</APIOptions>

<APIReturns type="Path | null">
  转换后的路径，如果路径被删除则返回 `null`。
</APIReturns>
</API>

### `ancestors`

获取给定路径的祖先路径列表。

<API name="ancestors">
<APIParameters>
  <APIItem name="path" type="Path">
    要获取祖先的路径。
  </APIItem>
  <APIItem name="options" type="PathAncestorsOptions" optional>
    祖先检索的选项。
  </APIItem>
</APIParameters>

<APIOptions type="PathAncestorsOptions">
  <APIItem name="reverse" type="boolean" optional>
    如果为 true，则按相反顺序返回路径（从最深到最浅）。
  </APIItem>
</APIOptions>

<APIReturns type="Path[]">
  按从最浅到最深的祖先排序的路径数组（除非反转）。
</APIReturns>
</API>

### `compare`

比较一个路径与另一个路径，返回一个整数，表示该路径是在另一个路径之前、相同位置还是之后。

<API name="compare">
<APIParameters>
  <APIItem name="path" type="Path">
    要比较的第一个路径。
  </APIItem>
  <APIItem name="another" type="Path">
    要比较的第二个路径。
  </APIItem>
</APIParameters>

<APIReturns type="-1 | 0 | 1">
  如果在之前则为 `-1`，如果在相同位置则为 `0`，如果在之后则为 `1`。
</APIReturns>
</API>

## 类型

表示索引数组，用于遍历文档树以到达特定节点。
