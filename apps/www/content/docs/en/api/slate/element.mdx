## Element API

### `isElementType`

Check if a value implements the `TElement` interface and has elementKey with selected value. Default it check to `type` key value

<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
  <APIItem name="elementVal" type="string">
    The value to match against.
  </APIItem>
  <APIItem name="elementKey" type="string" optional>
    The key to check. Defaults to 'type'.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is an element with matching type.
</APIReturns>

### `isAncestor`

Check if a value implements the 'Ancestor' interface.
<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is an ancestor node.
</APIReturns>

### `isElement`

Check if a value implements the `TElement` interface.

<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is an element.
</APIReturns>

### `isElementList`

Check if a value is an array of `TElement` objects.

<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is an array of elements.
</APIReturns>

### `isElementProps`

Check if a set of props is a partial of TElement.
<APIParameters>
  <APIItem name="props" type="any">
    The props to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the props match element properties.
</APIReturns>

### `matches`

Check if an element matches set of properties.

<APIParameters>
  <APIItem name="element" type="TElement">
    The element to check.
  </APIItem>
  <APIItem name="props" type="Partial<TElement>">
    The properties to match against.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the element matches all properties.
</APIReturns>

## Types

### `TElement`

`TElement` objects are a type of node in a Slate document that contain other element nodes or text nodes. They can be either "blocks" or "inlines" depending on the Slate editor's configuration.

<APIAttributes>
  <APIItem name="children" type="Descendant[]">
    An array of child nodes that can be either elements or text nodes.
  </APIItem>
  <APIItem name="type" type="string">
    A string identifier that defines the element's type (e.g., 'paragraph', 'heading', etc.).
  </APIItem>
  <APIItem name="[key: string]" type="unknown">
    Additional custom properties that can be added to the element.
  </APIItem>
</APIAttributes>

### `Element`

The `Element` interface is a type of node in a Slate document that contain other element nodes or text nodes. They can be either "blocks" or "inlines" depending on the Slate editor's configuration.

```ts
type Element = TElement;
```

### `ElementOrTextOf`

The `ElementOrTextOf` interface is a type that represents either an element or a text node from a specific editor type.

```ts
type ElementOrTextOf<E extends Editor> = ElementOf<E> | TextOf<E>;
```

### `ElementOrTextIn`

The `ElementOrTextIn` interface is a type that represents either an element or a text node from a specific value type.

```ts
type ElementOrTextIn<V extends Value> = ElementIn<V> | TextIn<V>;
```

### `ElementOf`

The `ElementOf` interface is a type that represents all the element nodes from a specific node type.

```ts
type ElementOf<N extends TNode> = Editor extends N
  ? TElement
  : TElement extends N
    ? TElement
    : N extends Editor
      ? ElementOf<N['children'][number]> | Extract<N['children'][number], TElement>
      : N extends TElement
        ? ElementOf<N['children'][number]> | Extract<N['children'][number], TElement> | N
        : never;
```

### `ElementIn`

The `ElementIn` interface is a type that represents an element from a specific value type.

```ts
type ElementIn<V extends Value> = ElementOf<V[number]>;
```
