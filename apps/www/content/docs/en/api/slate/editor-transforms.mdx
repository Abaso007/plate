---
title: Editor Transforms
description: API reference for editor transformation operations in Slate.
---

Transforms are helper functions operating on the document to modify nodes, text, marks, selection, and history.

## Node Operations

### `duplicateNodes`

Duplicates nodes from a specified location (or from the selection) and inserts them after that location.

<API name="duplicateNodes">

<APIOptions type="DuplicateNodesOptions">
  <APIItem name="...options" type="InsertNodesOptions" optional>
    `insertNodes` options.
  </APIItem>
  <APIItem name="at" type="At" optional>
    Location to get nodes from and insert after. Defaults to the selection.
  </APIItem>
  <APIItem name="block" type="boolean" optional>
    If `true`, duplicates the blocks above the location. Ignored if `nodes` is provided.
  </APIItem>
  <APIItem name="nodes" type="NodeEntry[]" optional>
    Specific nodes to duplicate. If provided, `block` is ignored.
  </APIItem>
</APIOptions>
</API>

### `insertFragment`

Insert a fragment of nodes at a location (defaults to the selection or end of the document).

<API name="insertFragment">
<APIParameters>
  <APIItem name="fragment" type="N[]">
    The fragment of nodes to insert.
  </APIItem>
  <APIItem name="options" type="InsertFragmentOptions" optional />
</APIParameters>

<APIOptions type="InsertFragmentOptions">
  <APIItem name="at" type="At" optional>
    Location to insert. Defaults to current selection.
  </APIItem>
  <APIItem name="batchDirty" type="boolean" optional>
    Whether to batch dirty operations.
  </APIItem>
  <APIItem name="hanging" type="boolean" optional>
    Whether the range is hanging.
  </APIItem>
  <APIItem name="voids" type="boolean" optional>
    Whether to allow insertion inside void nodes.
  </APIItem>
</APIOptions>
</API>

<API name="insertNode">
<APIParameters>

    The node to insert.
  </APIItem>
  <APIItem name="options" type="InsertNodesOptions" optional />
</APIParameters>
</API>

### `insertNodes`

Insert one or more nodes atomically at a location (or selection/end of the document if not defined).

<API name="insertNodes">
<APIParameters>
  <APIItem name="nodes" type="N | N[]">
    A single node or an array of nodes to insert.
  </APIItem>
  <APIItem name="options" type="InsertNodesOptions" optional />
</APIParameters>

<APIOptions type="InsertNodesOptions">
  <APIItem name="...options" type="QueryOptions" optional>
    Common query options.
  </APIItem>
  <APIItem name="batchDirty" type="boolean" optional>
    Whether to batch dirty operations.
  </APIItem>
  <APIItem name="hanging" type="boolean" optional>
    Whether the insertion is hanging.
  </APIItem>
  <APIItem name="nextBlock" type="boolean" optional>
    Insert after the current block if the current block is removed via `removeEmpty`.
  </APIItem>
  <APIItem name="removeEmpty" type="QueryNodeOptions | boolean" optional>
    Remove the current block if empty. Defaults to removing an empty paragraph; can be customized.
  </APIItem>
  <APIItem name="select" type="boolean" optional>
    If `true`, selects the inserted nodes.
  </APIItem>
  <APIItem name="voids" type="boolean" optional>
    Allow insertion into void nodes.
  </APIItem>
</APIOptions>
</API>

Set properties on nodes.

<API name="setNodes">

  </APIItem>
  <APIItem name="hanging" type="boolean" optional>
    Whether the operation is hanging.
  </APIItem>
  <APIItem name="marks" type="boolean" optional>
    When `true`, only apply to text nodes in non-void or markable void nodes.
  </APIItem>
  <APIItem name="merge" type="(prop: Partial<Descendant>, node: Partial<Descendant>) => object" optional>
    Custom function to merge properties between nodes.
  </APIItem>
  <APIItem name="mode" type="'highest' | 'lowest'" optional>
    The matching mode to use when finding nodes.
  </APIItem>
  <APIItem name="split" type="boolean" optional>
    Whether to allow splitting nodes when setting properties.
  </APIItem>
  <APIItem name="voids" type="boolean" optional>
    Whether to allow modifications in void nodes.
  </APIItem>
</APIOptions>
</API>

## Mark Operations

### `addMark`

Add a mark with a certain key and value to text nodes in the selection.

<API name="addMark">
<APIParameters>
  <APIItem name="key" type="string">
    The mark key.
  </APIItem>
  <APIItem name="value" type="any">
    The value for that mark.
  </APIItem>
</APIParameters>
</API>

### `addMarks`

Add multiple marks, optionally removing certain marks first.

```ts
editor.tf.addMarks({ bold: true, italic: true });
editor.tf.addMarks({ bold: subscript }, { remove: 'superscript' });
editor.tf.addMarks({ bold: true }, { remove: ['italic', 'underline'] });
```

<API name="addMarks">
<APIParameters>
  <APIItem name="marks" type="Record<string, any>">
    Marks key-value pairs.
  </APIItem>
  <APIItem name="options" type="AddMarksOptions" optional />
</APIParameters>

<APIOptions type="AddMarksOptions">
  <APIItem name="remove" type="string[] | string" optional>
    Mark keys to remove first (useful for mutually exclusive marks).
  </APIItem>
</APIOptions>
</API>

### `removeMark`

Remove a mark from text nodes in the selection (or from `editor.marks` if selection is collapsed).

<API name="removeMark">
<APIParameters>
  <APIItem name="key" type="string">
    The mark key to remove.
  </APIItem>
</APIParameters>
</API>

### `removeMarks`

Remove specific marks (or all marks if keys are not specified) from nodes in the selection.

```ts
editor.tf.removeMarks()             // remove all marks
editor.tf.removeMarks('bold')       // remove the 'bold' mark
editor.tf.removeMarks(['bold','italic'])
editor.tf.removeMarks('bold', { at: range })
```

<API name="removeMarks">
<APIParameters>
  <APIItem name="keys" type="string | string[]" optional>
    Mark keys to remove. If not provided, removes all marks from `editor.marks`.
  </APIItem>
  <APIItem name="options" type="RemoveMarksOptions" optional />
</APIParameters>

<APIOptions type="RemoveMarksOptions">
  <APIItem name="...options" type="UnsetNodesOptions" optional>
    `unsetNodes` options.
  </APIItem>
  <APIItem name="at" type="TRange" optional>
    Range where the mark(s) will be removed. Defaults to current selection.
  </APIItem>
  <APIItem name="shouldChange" type="boolean" optional>
    If `true`, force an `onChange` call even if the selection is collapsed.
  </APIItem>
</APIOptions>
</API>

### `toggleMark`

Toggle a mark on or off in the selection.  
Optionally remove certain marks first when toggling on (e.g., removing `superscript` before adding `subscript`).

```ts
editor.tf.toggleMark('bold');
editor.tf.toggleMark('subscript', { remove: 'superscript' });
```

<API name="toggleMark">
<APIParameters>
  <APIItem name="key" type="string">
    The mark key to toggle.
  </APIItem>
  <APIItem name="value" type="any" optional>
    Optional new value for the toggled mark.
  </APIItem>
  <APIItem name="options" type="ToggleMarkOptions" optional />
</APIParameters>

<APIOptions type="ToggleMarkOptions">
  <APIItem name="remove" type="string[] | string" optional>
    Mark keys to remove first when toggling on.
  </APIItem>
</APIOptions>
</API>

## Selection

### `collapse`

Collapse the selection to one edge.

<API name="collapse">

<APIOptions type="object">
  <APIItem name="edge" type="'anchor' | 'focus' | 'start' | 'end'" optional>
    Where to collapse: start or end of selection, anchor, or focus.
  </APIItem>
</APIOptions>
</API>

### `deselect`

Unset the selection.

### `move`

Move the selection’s anchor/focus by a certain distance and unit.

<API name="move">

<APIOptions type="object">
  <APIItem name="distance" type="number" optional>
    Number of units to move. Defaults to 1.
  </APIItem>
  <APIItem name="unit" type="'offset' | 'character' | 'word' | 'line'" optional>
    The unit to move by. Defaults to `'character'`.
  </APIItem>
  <APIItem name="reverse" type="boolean" optional>
    Whether to move backward instead of forward.
  </APIItem>
  <APIItem name="edge" type="'anchor' | 'focus' | 'start' | 'end'" optional>
    Which edge of the selection to move.
  </APIItem>
</APIOptions>
</API>

### `select`

Select a location (`At`). Supports options like `edge` (start/end), `focus`, or sibling navigation (`next`, `previous`).

```ts
editor.tf.select(at)
editor.tf.select(at, { edge: 'end' })
editor.tf.select(at, { edge: 'start' })
```

<API name="select">
<APIParameters>
  <APIItem name="at" type="At">
    Location to select.
  </APIItem>
  <APIItem name="options" type="SelectOptions" optional />
</APIParameters>

<APIOptions type="SelectOptions">
  <APIItem name="edge" type="'start' | 'end'" optional>
    Select the start or end edge above `at`.
  </APIItem>
  <APIItem name="focus" type="boolean" optional>
    Focus the editor first.
  </APIItem>
  <APIItem name="next" type="boolean" optional>
    Select the start of the next sibling.
  </APIItem>
  <APIItem name="previous" type="boolean" optional>
    Select the end of the previous sibling.
  </APIItem>
</APIOptions>
</API>

### `setPoint`

Set properties on a point (anchor/focus/start/end).

<API name="setPoint">
<APIParameters>
  <APIItem name="props" type="Partial<Point>">
    Point properties to update.
  </APIItem>
  <APIItem name="options" type="object" optional />
</APIParameters>

<APIOptions type="object">
  <APIItem name="edge" type="'anchor' | 'focus' | 'start' | 'end'" optional>
    Which edge of the selection to update.
  </APIItem>
</APIOptions>
</API>

### `setSelection`

Set or update the properties of the existing selection.  
If no selection exists, this method does nothing—use `select` to create one.

<API name="setSelection">
<APIParameters>
  <APIItem name="props" type="Partial<TRange>">
    New selection properties to update.
  </APIItem>
</APIParameters>
</API>
