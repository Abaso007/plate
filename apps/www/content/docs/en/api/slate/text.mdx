---
title: Text
description: API reference for text nodes in Slate.
---

`Text` objects represent the nodes that contain the actual text content of a Slate document along with any formatting properties. They are always leaf nodes in the document tree as they cannot contain any children.

## Text API

### `decorations`

Get the leaves for a text node given decorations.

<APIParameters>
  <APIItem name="node" type="TText">
    The text node to get leaves from.
  </APIItem>
  <APIItem name="decorations" type="DecoratedRange[]">
    Array of decorated ranges to apply.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `TText[]` - An array of text nodes with the decorations applied.
</APIReturns>

### `equals`

Check if two text nodes are equal.

<APIParameters>
  <APIItem name="text" type="TText">
    First text node to compare.
  </APIItem>
  <APIItem name="another" type="TText">
    Second text node to compare.
  </APIItem>
  <APIItem name="options" type="TextEqualsOptions" optional>
    <APISubList>
      <APISubListItem parent="options" name="loose" type="boolean" optional>
        If true, the text content is not compared. This is used to check whether sibling text nodes can be merged.
      </APISubListItem>
    </APISubList>
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the text nodes are equal according to the comparison rules, `false` otherwise.
</APIReturns>

### `isText`

Check if a value implements the `Text` interface.

<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is a valid text node, `false` otherwise.
</APIReturns>

### `isTextList`

Check if a value is a list of `Text` objects.

<APIParameters>
  <APIItem name="value" type="any">
    The value to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the value is an array of text nodes, `false` otherwise.
</APIReturns>

### `isTextProps`

Check if some props are a partial of Text.

<APIParameters>
  <APIItem name="props" type="any">
    The props to check.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the props match a partial text node structure, `false` otherwise.
</APIReturns>

### `matches`

Check if a text matches set of properties.

<APIParameters>
  <APIItem name="text" type="TText">
    The text node to check.
  </APIItem>
  <APIItem name="props" type="Partial<TText>">
    The properties to match against.
  </APIItem>
</APIParameters>

<APIReturns>
  Returns `boolean` - `true` if the text node matches the properties, `false` otherwise.
</APIReturns>

## Types

### `TText`

`TText` objects represent the nodes that contain the actual text content of a Slate document along with any formatting properties. They are always leaf nodes in the document tree as they cannot contain any children.

<APIAttributes>
  <APIItem name="text" type="string">
    The text content of the node.
  </APIItem>
  <APIItem name="[key: string]" type="unknown">
    Additional formatting properties that can be added to the text node.
  </APIItem>
</APIAttributes>

### `Text`

A concrete implementation of the `TText` type. Text objects represent the nodes that contain the actual text content of a Slate document along with any formatting properties. They are always leaf nodes in the document tree as they cannot contain any children.

### `DecoratedRange`

A range object that includes decoration information. Used to apply formatting or styling to specific ranges of text within a document.

### `TextEqualsOptions`

Options for comparing text nodes.

<APIAttributes>
  <APIItem name="loose" type="boolean" optional>
    If true, the text content is not compared. This is used to check whether sibling text nodes can be merged based only on their formatting properties.
  </APIItem>
</APIAttributes>

### `TextOf<N>`

A utility type that extracts all possible text node types from a root node type. This is useful when you need to work with text nodes in a strongly-typed way within your editor implementation.

<APIParameters>
  <APIItem name="N" type="TNode">
    The root node type to extract text types from.
  </APIItem>
</APIParameters>

### `TextIn<V>`

A utility type that extracts all text node types from a Value type. This is similar to `TextOf` but works specifically with editor Value types.

<APIParameters>
  <APIItem name="V" type="Value">
    The Value type to extract text types from.
  </APIItem>
</APIParameters>

### `MarksOf<N>`

A utility type that extracts all possible mark types from a root node type. Marks are the formatting properties that can be applied to text nodes.

<APIParameters>
  <APIItem name="N" type="TNode">
    The root node type to extract mark types from.
  </APIItem>
</APIParameters>

### `MarksIn<V>`

A utility type that extracts all mark types from a Value type. This is similar to `MarksOf` but works specifically with editor Value types.

<APIParameters>
  <APIItem name="V" type="Value">
    The Value type to extract mark types from.
  </APIItem>
</APIParameters>

### `MarkKeysOf<N>`

A utility type that extracts all possible mark property keys from a node type. This is useful when you need to work with mark properties in a strongly-typed way.

<APIParameters>
  <APIItem name="N" type="TNode">
    The node type to extract mark keys from.
  </APIItem>
</APIParameters>
